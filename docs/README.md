# Document Control

This folder treats the repository as a single project and contains the *control artifacts* used to plan, track, review, and report on changes.

## What goes here

- `open_items_register.md`: the single canonical register for blockers/risks/issues/assumptions/dependencies
- `specs/`: design docs and specifications for planned changes
- `plans/`: implementation plans derived from approved specs
- `findings/`: review/audit findings (framework reviews, spec reviews, gap analyses)
- `reports/`: status reports, assessment reports, release notes, summaries
- `session_digests/`: LoreSpec session digests generated from conversations (see `AGENTS.md`)
